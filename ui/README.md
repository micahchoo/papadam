# papadam/ui

SvelteKit 2 + Svelte 5 SPA — papadam frontend.

See [`../ARCHITECTURE.md`](../ARCHITECTURE.md) for the full system design.

---

## Local development

```bash
npm install

# Copy and fill in the backend URLs
cp .env.example .env.local
# VITE_API_URL=http://localhost:8000
# VITE_CRDT_URL=ws://localhost:1234

npm run dev           # dev server on :5173
```

`.env.local` is gitignored by Vite automatically.
In production the SPA fetches `/config.json` from Django instead of using env vars.

---

## Commands

```bash
npm run dev           # Vite dev server
npm run build         # production build → build/ (flat static files)
npm run preview       # preview production build locally

npm run check         # svelte-check + TypeScript
npm run lint          # ESLint (includes architecture boundary enforcement)
npm test              # Vitest unit tests (once)
npm run test:watch    # Vitest watch mode
npm run test:e2e      # Playwright end-to-end tests
npm run test:all      # vitest run + playwright
```

## Build output

`npm run build` produces a flat static site in `build/`:

```
build/
├── index.html          ← SPA entry point (nginx try_files fallback target)
├── service-worker.js   ← Workbox precache + runtime cache + Background Sync
├── _app/               ← hashed JS + CSS bundles
├── img/                ← static images
└── favicon.png
```

This directory is mounted into the `papadam-ui` nginx container in production.

## Tech stack

| Layer          | Choice                                                |
| -------------- | ----------------------------------------------------- |
| Framework      | SvelteKit 2, Svelte 5 rune mode                       |
| Language       | TypeScript (strict)                                   |
| Styling        | Tailwind CSS                                          |
| Components     | bits-ui 0.21 (headless, accessible)                   |
| Media player   | HLS.js (audio + video, adaptive bitrate)              |
| CRDT / offline | Y.js + y-indexeddb + y-websocket                      |
| Service worker | Workbox (precache, runtime cache, Background Sync)    |
| i18n           | Paraglide (message catalog in `messages/`)            |
| Unit tests     | Vitest + @testing-library/svelte                      |
| E2E tests      | Playwright (chromium + firefox + Pixel 5 mobile)      |
| Linting        | ESLint (typescript-eslint + eslint-plugin-boundaries) |
| Build          | Vite + adapter-static                                 |

## Module boundary rules (eslint-plugin-boundaries)

```
api.ts        leaf — pure HTTP client, no internal imports
events.ts     → api only
crdt.ts       → stores only
config.ts     → api only
stores.ts     → config, crdt
i18n/         leaf (generated by Paraglide)
primitives/   leaf — unstyled accessible component wrappers
components/   → api, stores, config, events, i18n, primitives
routes/       → anything in lib/
```

Violations fail `npm run lint` and the CI lint step.

## Runtime config

The SPA resolves backend URLs at startup:

1. If `VITE_API_URL` is set (local dev / `.env.local`) — use it directly, skip fetch.
2. Otherwise fetch `GET /config.json` from Django — returns `{"API_URL": "...", "CRDT_URL": "..."}`.

Set in `src/lib/config.ts`. Called once from `+layout.svelte` on mount.
